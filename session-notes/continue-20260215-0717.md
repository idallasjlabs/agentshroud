- "therealidallasj" is the AI agent/bot that represents Isaiah
- This naming is critical - user was frustrated when UI still showed "Isaiah"

### Why Docker Secrets Not Environment Variables?
- Security best practice - secrets not visible in `docker inspect`
- IEC 62443 compliance requirement (FR4 - Data Confidentiality)
- Secrets read from `/run/secrets/` tmpfs mount inside container
- OpenAI API key stored at: `/run/secrets/openai_api_key` (inside openclaw container)

### Why Read-Only Root Filesystem?
- IEC 62443 FR3 requirement (System Integrity)
- Prevents runtime container modification
- Required PYTHONDONTWRITEBYTECODE=1 env var to avoid .pyc write attempts
- Requires explicit tmpfs mounts for `/tmp`, `/var/tmp`, Python cache dirs

### Architecture Decision: Internal Network with Internet Access
- `docker-compose.yml` line 137: `internal: false` (not fully isolated)
- Reason: OpenClaw container needs HTTPS access to `api.openai.com`
- Gateway is localhost-only (127.0.0.1:8080), provides network security boundary
- Full network isolation deferred to Phase 5 (would require proxy through gateway)

### File Extension Gotcha
- Persona files mounted without .md extension: `IDENTITY` and `USER` (not `IDENTITY.md`, `USER.md`)
- Only `SOUL.md` has .md extension
- This caused persona loading to fail initially - code was looking for wrong filenames
- Root cause: docker-compose.yml volume mounts specify target names explicitly
