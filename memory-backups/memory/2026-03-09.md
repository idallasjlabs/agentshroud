### Trillian ✅ (fixed glibc TLS)
- **Root cause found:** Custom seccomp profiles (`gateway-seccomp.json`, `agentshroud-seccomp.json`) block `arch_prctl` syscall needed for glibc TLS allocation on x86_64
- ARM (Marvin/Pi) uses a different TLS mechanism, so it worked there
- NOT a base image issue — tested `docker run` vs `docker-compose` to isolate
- **Fix:** Added `security_opt: !override` in Trillian's compose override to drop custom seccomp, keep `no-new-privileges:true`
- Also bumped gateway mem_limit from 512m to 1g
- Gateway + bot both running after fix
- Same 1Password session issue as Pi
- **Action item:** Update seccomp profiles to allow `arch_prctl` on x86_64 (proper fix vs override)
