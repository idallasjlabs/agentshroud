# AgentShroud Brand Guidelines

## Brand Identity

### Mission
AgentShroud provides enterprise-grade security for AI agents, enabling safe autonomous operation through PII redaction, approval workflows, and comprehensive audit trails.

### Vision
To become the trusted security layer for AI agent deployments, protecting sensitive data while enabling innovation.

### Values
- **Security First** - Never compromise on data protection
- **Transparency** - Open-source approach, clear audit trails
- **Privacy** - PII redaction at the core
- **Trust** - Enterprise-grade reliability
- **Innovation** - Advanced AI safety features

## Brand Personality

### Voice & Tone

**Professional but Approachable**
- Technical accuracy without jargon
- Clear, concise communication
- Confident but not arrogant

**Security-Focused**
- Emphasize protection and safety
- Highlight compliance and standards
- Reference security certifications

**Developer-Friendly**
- Speak the language of developers
- Provide clear technical documentation
- Share code examples and best practices

### Writing Style

**Do:**
- Use active voice
- Be specific and concrete
- Provide examples
- Explain complex concepts simply

**Don't:**
- Use marketing fluff
- Make unsubstantiated claims
- Over-promise capabilities
- Use fear-based messaging

## Logo Usage

### Clear Space
Maintain minimum clear space of 50% of logo height on all sides.

### Minimum Size
- **Digital:** 120px width minimum
- **Print:** 1 inch width minimum

### Backgrounds

**Light Backgrounds**
- Use primary logo with transparent background
- Ensure sufficient contrast (4.5:1 minimum)

**Dark Backgrounds**
- Use white or light variant
- Test visibility across different displays

**Colored Backgrounds**
- Use transparent logo
- Ensure contrast meets accessibility standards

### Prohibited Uses
- ❌ Stretching or distorting
- ❌ Rotating at non-standard angles
- ❌ Applying gradients or effects
- ❌ Changing colors
- ❌ Placing on busy backgrounds
- ❌ Using low-resolution files

## Color Usage

### Primary Color Application
Use AgentShroud Blue (`#1583f0`) for:
- Primary CTAs
- Interactive elements
- Logo and brand accents
- Navigation highlights
- Active states

### Accessibility
All color combinations must meet WCAG 2.1 AA standards:
- **Normal text:** 4.5:1 contrast ratio minimum
- **Large text:** 3:1 contrast ratio minimum
- **Interactive elements:** 3:1 against adjacent colors

## Typography in Practice

### Hierarchy
Maintain clear visual hierarchy:
1. Headlines grab attention
2. Subheads organize content
3. Body text provides detail
4. Labels clarify function

### Readability
- **Line length:** 50-75 characters optimal
- **Paragraph spacing:** 1.5x line height
- **Alignment:** Left-aligned for body text
- **Orphans/Widows:** Avoid when possible

## Co-Branding Guidelines

### Principles

When representing AgentShroud™ alongside another brand:
1. **Equal prominence** — neither brand dominates unless the context is clearly partner-hosted
2. **Separation** — maintain clear visual separation (minimum 2× logo height) between the two logos
3. **Authenticity** — use only official AgentShroud™ assets; never recreate the logo
4. **Approval required** — all co-branded materials require written approval from Isaiah Dallas Jefferson, Jr.

### Placement Rules

**Horizontal lockup:**
`[Partner Logo]  ·  [AgentShroud™ Logo]`

- Use the transparent PNG logo on partner-colored backgrounds
- Use the dark-background PNG when the partner logo is on white/light
- Minimum clear space: 50% of AgentShroud™ logo height on all sides

**Powered-by / Integration badge:**
Use `badge-120x120.png` with the text "Powered by AgentShroud™" or "AgentShroud™ Integrated" — never modify the badge itself.

### Color Constraints
- Do not place the AgentShroud™ logo on backgrounds that conflict with `#1583f0`
- Do not use a partner's brand color as a fill for the AgentShroud™ logo
- On colored backgrounds, ensure ≥ 3:1 contrast between the logo and background

### Prohibited Uses
- ❌ Implying endorsement of a partner's product without written agreement
- ❌ Using AgentShroud™ name or logo in competitive comparisons
- ❌ Altering the AgentShroud™ logo to incorporate partner colors or marks
- ❌ Creating derivative logos or wordmarks that combine both brands into one mark
- ❌ Using co-branded materials before written approval is obtained

### Approval Process
1. Submit use case description to agentshroud.ai@gmail.com
2. Attach mockup of proposed co-branded material
3. Receive written approval before publishing
4. Re-approval required for any material change in scope or design

## Animation Guidelines

### Motion Principles

AgentShroud™ motion design reflects the brand values of precision, security, and transparency:
- **Purposeful** — every animation communicates meaning; no decorative movement
- **Efficient** — transitions are brief and do not interrupt user flow
- **Calm** — no aggressive or attention-grabbing motion; the product is a governance layer, not entertainment
- **Accessible** — all animations respect `prefers-reduced-motion`

### Timing & Easing

| Token | Value | Use |
|-------|-------|-----|
| `--duration-fast` | `120ms` | Micro-interactions (toggles, focus rings) |
| `--duration-base` | `180ms` | State transitions (hover, active) |
| `--duration-slow` | `300ms` | Panel open/close, modal entrance |
| `--duration-page` | `400ms` | Page-level transitions |
| `--ease-standard` | `cubic-bezier(0.4, 0, 0.2, 1)` | Default — Material-style ease |
| `--ease-decelerate` | `cubic-bezier(0, 0, 0.2, 1)` | Elements entering the screen |
| `--ease-accelerate` | `cubic-bezier(0.4, 0, 1, 1)` | Elements leaving the screen |

### CSS Reference

```css
/* Standard transition — apply via var(--transition) */
:root {
  --transition: all 180ms cubic-bezier(0.4, 0, 0.2, 1);
}

/* Reduce motion — always include this override */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

### Logo Animation Constraints

When animating the AgentShroud™ logo:
- **Do:** Fade-in on load (opacity 0 → 1, 300ms decelerate)
- **Do:** Subtle scale entrance (scale 0.97 → 1.0, 300ms decelerate)
- **Do not:** Rotate, spin, bounce, or distort the logo
- **Do not:** Apply color-shift or glow animations to the logo itself
- **Do not:** Loop the logo animation — it should play once on load only

### Status & Feedback Animations

The design system provides two named keyframes for status indicators:

```css
@keyframes pulse-success {
  0%, 100% { opacity: 1; }
  50%       { opacity: 0.5; }
}

@keyframes pulse-danger {
  0%, 100% { opacity: 1; }
  50%       { opacity: 0.3; }
}
```

Use `pulse-success` (2s infinite) for live/healthy indicators and `pulse-danger` (2s infinite) for critical alerts. Warning states do not pulse — they use static amber.

### Accessibility

- Always implement `prefers-reduced-motion` overrides (see CSS Reference above)
- Never use animation as the sole indicator of state — pair with color, icon, or text
- Flashing content must stay below 3 flashes per second (WCAG 2.1 guideline 2.3.1)
