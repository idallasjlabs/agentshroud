10 Skills/Scripts for Your Repo: Auto-Deploy OpenClaw in a Secure Docker Container (No LAN, VPN-Only, Internet Access)
1. Hardened docker-compose.yml with Network Isolation — Create an internal Docker network where OpenClaw is only on the internal network, and route internet access through a proxy container on both internal and external networks. Substack Use network: none on the sandbox containers, and a dedicated proxy (LiteLLM or Squid) for outbound API calls only.
2. Rootless Podman Alternative Script — Podman runs without a daemon and without root privileges; if an attacker escapes the container, they land as an unprivileged user. Docker runs a daemon as root, so a container escape means root on the host. Substack Include a Podman-based deployment option as the safer default.
3. Container Hardening Baseline Script — Automate the security primitives in your entrypoint: cap_drop: ALL, no-new-privileges: true, readOnlyRoot: true, tmpfs on /tmp and /var/tmp, pidsLimit: 256, memory limits, and a custom seccomp profile. These should be non-negotiable defaults.
4. VPN-Only Networking with iptables Lockdown — Create a startup script that configures the host's firewall to drop all LAN traffic to/from the OpenClaw container while allowing only traffic through your Tailscale/WireGuard interface. Given your existing Tailscale infrastructure, bind the gateway to the Tailscale IP only, not LAN or 0.0.0.0.
5. Reverse Proxy with Auth (Nginx/Caddy + TLS) — Include a pre-configured nginx or Caddy container that sits in front of the gateway with auto-TLS, token-based auth, and rate limiting. The default setup exposes your Gateway to the public internet with no auth. Macaron Your script should enforce this by default.
6. Credential Injection via Docker Secrets — Script that provisions API keys (Anthropic, messaging platform tokens) through Docker secrets or mounted encrypted volumes rather than environment variables or config files. Secrets should be rotated on a 90-day schedule with automated reminders.
7. openclaw security audit CI/CD Integration — Add a GitHub Actions workflow to your repo that runs the built-in security audit on every config change and PR. It flags common footguns like gateway auth exposure, elevated allowlists, and loose filesystem permissions, and --fix applies safe guardrails. Openclaw
8. ClawHub Skill Scanner Pre-Install Gate — Script that automatically runs skills through Cisco's Skill Scanner (or the SafeClaw scanner) before allowing installation. Checks for data exfiltration patterns, backdoor commands, obfuscated code, and excessive permission requests. No skill gets installed without passing the scan.
9. Log Aggregation & Anomaly Detection Stack — Include a lightweight logging sidecar (Promtail/Loki or Fluent Bit) that captures skill execution logs, outbound network connections, and model API usage. Ship these to your existing monitoring infrastructure. Given your Zabbix setup, you could create custom triggers for suspicious patterns.
10. Automated Health Check & Drift Detection Script — A cron-based script (or Docker healthcheck) that continuously verifies your security posture: confirms containers aren't running as root, checks that the gateway is bound to localhost/Tailscale only, validates file permissions on ~/.openclaw (700) and config (600), confirms network: none on sandbox containers, and alerts if any setting has drifted from baseline.
