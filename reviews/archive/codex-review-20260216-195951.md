[HIGH] gateway/ingest_api/sanitizer.py:339–346 — Missing AWS-style access-key detection  
The test suite includes a case for blocking AWS access keys (`Access key: AKIAIOSFODNN7EXAMPLE`), but the credential‐blocking patterns only look for generic `api[_\s]?key`, `secret`, `token`, TOTP, high‐entropy strings with special characters, SSH private keys, and credit-card numbers. AWS access keys (which start with `AKIA` followed by 16 uppercase alphanumerics) won’t match any of these patterns and will slip through unredacted.  
  Suggested fix: add a dedicated regex for AWS IAM access keys (and similarly for secret access keys), for example:  
  ```python
  # in __init__
  self._aws_key_pattern = re.compile(r'\bAKIA[0-9A-Z]{16}\b')
  …
  credential_patterns = [
      …
      (self._aws_key_pattern, "aws_access_key"),
      # also consider r'[0-9a-zA-Z/+]{40}' for AWS secret key patterns
      …
  ]
  ```  
[MEDIUM] gateway/ingest_api/config.py:38–46 — Unvalidated `targets` URLs allow SSRF  
The new `RouterConfig` validates only the `default_url` field, but any additional targets loaded into `config.targets: dict[str,str]` are not checked. A malicious or typo’d entry in your YAML could direct the gateway to forward user data to an internal metadata service (e.g. `http://169.254.169.254/`) or an arbitrary external host.  
  Suggested fix: apply the same Pydantic `field_validator` to each URL in `targets`, or change the type of `targets` to `dict[str, HttpUrl]` so that Pydantic enforces both scheme and hostname allow‐list checks on every entry.  
[MEDIUM] scripts/gemini-review.py:24 — API key exposed in query parameter  
The Gemini HTTP request appends `?key=$GEMINI_API_KEY` to the URL. Query strings may be logged by intermediate proxies or recorded in server access logs, inadvertently exposing your secret.  
  Suggested fix: move the API key into an `Authorization: Bearer <key>` header instead of the URL, and drop it from the query string. For example:  
  ```python
  req = Request(
      API_URL,
      data=payload,
      headers={
          "Content-Type": "application/json",
          "Authorization": f"Bearer {API_KEY}",
      },
  )
  ```
