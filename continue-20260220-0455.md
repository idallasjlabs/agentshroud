## Key Decisions & Context

### Decision: Symlink ~/.openclaw → ~/.agentshroud
- **Why:** OpenClaw is an npm package we depend on (installed globally in container)
- The CLI hardcodes `~/.openclaw` as its config directory
- We want to brand as AgentShroud but can't change OpenClaw's internals
- Symlink allows both paths to work, maintaining OpenClaw CLI compatibility

### Decision: Use OPENCLAW_* env vars despite rebrand
- **Why:** OpenClaw CLI expects specific env var names (OPENCLAW_GATEWAY_PASSWORD, etc.)
- We can't override these without forking OpenClaw
- Trade-off: Keep OpenClaw env var names for compatibility, brand everything else as AgentShroud

### Decision: Execute `openclaw gateway` command despite rebrand
- **Why:** We install the `openclaw` npm package globally
- There is no `agentshroud` binary - it's a conceptual wrapper/rebrand
- The actual CLI command remains `openclaw`

### Gotcha: Port Conflict on 18789
- docker-compose.yml maps 127.0.0.1:18790→18789 (not 18789→18789)
- Reason: Port conflict during development, needs restart to reclaim 18789
