## 1. SOC Command Center (Web UI)

### Access and Login

```
URL: http://localhost:9080/soc/
```

Read the gateway password:

```bash
cat docker/secrets/gateway_password.txt
```

Enter that value at the SOC login prompt. A session cookie is set on success.

### Dashboard Tabs to Verify

#### Security Tab
- Event list loads (empty on fresh start is fine)
- "Export Audit" button → downloads a CEF log file
- Risk Score badge is visible

#### Egress Tab
- Shows "No pending egress requests" on fresh start
- **Test:** have a collaborator send a message that triggers a web fetch (ask the bot to look something up online)
- An egress approval card should appear — click Approve or Deny and confirm the bot responds accordingly

#### Users Tab
- All collaborators from `agentshroud.yaml` appear with their roles (COLLABORATOR, ADMIN, etc.)
- Click a user → shows role, group membership, last active timestamp
- "Add Collaborator" → enter a Telegram user ID → confirm it persists after page refresh

#### Groups Tab
- Groups `sort`, `gsdea`, `gsde` appear with member counts
- Click a group → shows members, admin, collab_mode, assigned projects
- Change a group's collab_mode → save → refresh → confirm it changed

#### Scanners Tab
- Shows Falco, ClamAV, Wazuh, Trivy status cards
- On first run, cards will show `unknown` status until sidecars produce findings
- Run to populate: `bash scripts/security-scan.sh --skip-cosign`

#### Scorecard Tab
- 12-domain IEC 62443 / CIS / NIST maturity grid (0–5 scale)
- Populated after `security-scan.sh` runs

### API Endpoints — Direct Verification

Set your token once:

```bash
TOKEN=$(cat docker/secrets/gateway_password.txt)
```

**Delegation:**

```bash
# List active delegations (should be empty initially)
curl -s -H "Authorization: Bearer $TOKEN" \
  http://localhost:9080/soc/v1/delegation | jq .

# Create a delegation — give Keith egress approval rights for 8 hours
curl -s -X POST -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"delegatee_id":"8279589982","privilege":"egress_approval","duration_hours":8}' \
  http://localhost:9080/soc/v1/delegation | jq .

# Confirm it appears
curl -s -H "Authorization: Bearer $TOKEN" \
  http://localhost:9080/soc/v1/delegation | jq .

# Revoke it
curl -s -X DELETE -H "Authorization: Bearer $TOKEN" \
  "http://localhost:9080/soc/v1/delegation/8279589982?privilege=egress_approval" | jq .

# Confirm it's gone
curl -s -H "Authorization: Bearer $TOKEN" \
  http://localhost:9080/soc/v1/delegation | jq .
```

## 3. Collaborator Experience Testing

Have real collaborators message the bot directly on Telegram. Observe results from the owner side.

### Test: local_only mode

```
/setmode sort local_only
```

Have **Keith** send any message to the bot.

Expected: Keith receives a local info response ("You have collaborator access…") — the AI is never called. You see no forwarded message in the bot logs.

### Test: project_scoped mode

```
/setmode sort project_scoped
```

Have **Keith** send these two messages:

1. `How do I configure a Zabbix trigger for battery voltage alarms?`
   - Expected: **reaches the AI** (on-topic — SORT/fleet-alarms scope)
   - Bot responds with AI-generated answer

2. `What's a good recipe for pasta?`
   - Expected: **blocked locally** with "This message is outside your current project scope."

### Test: full_access mode

```
/setmode sort full_access
```

Have **Keith** send any message — should always reach the AI regardless of topic.

Restore afterward:

```
/setmode sort project_scoped
```

### Test: Delegation privilege enforcement

1. Delegate egress approval to Keith:
   ```
   /delegate keith egress_approval 1h
   ```
2. Have the bot attempt a web fetch (trigger an egress request from bot activity)
3. Confirm Keith can approve the egress request from his Telegram session
4. Revoke the delegation:
   ```
   /revoke-delegation keith egress_approval
   ```
5. Trigger another egress request — confirm Keith can no longer approve it

## 6. Security Scan

Run the build-time security scanner to populate the Scorecard and Scanners tabs:

```bash
bash scripts/security-scan.sh --skip-cosign
```

Reports written to `reports/` directory:
- `reports/trivy/` — CVE scan results
- `reports/sbom/` — SPDX JSON software bill of materials
- `reports/openscap/` — CIS compliance scan (requires `oscap-docker`)
- `reports/semgrep/` — SAST findings

After the scan, refresh the SOC dashboard Scorecard tab to see updated maturity scores.
