# v0.8.0 Feature: Interactive Egress Firewall ("Little Snitch for Agents")

## Concept
All outbound network traffic requires explicit human approval. Like Little Snitch on macOS, but for AI agent connections.

## User Experience
When an agent tries to reach a new domain:
1. **Prompt owner** via Telegram/chat with:
   - Domain name (e.g., `api.stripe.com:443`)
   - Service/tool requesting access (e.g., `web_fetch`, `http_request`)
   - Agent ID making the request
   - Risk assessment: 🟢 Green (known safe), 🟡 Yellow (unknown), 🔴 Red (suspicious)
2. **Owner chooses**:
   - ✅ Allow permanently (add to allowlist)
   - ⏱️ Allow temporarily (until session end / restart / N minutes)
   - 🚫 Deny permanently (add to denylist)
   - ❌ Deny once

## Console Management
- View all allowed/denied domains with approval type and timestamp
- Revoke permanent approvals
- See recent connection attempts (allowed + denied)
- Per-agent allowlist views
- Bulk approve/deny

## Architecture (high-level)
- Builds on existing `EgressFilter` + `EgressFilterConfig` from v0.7.0
- New: `EgressApprovalQueue` — holds pending requests, notifies owner
- New: `/manage/egress/rules` — CRUD for firewall rules
- New: `/manage/egress/pending` — pending approval requests
- New: Telegram inline buttons for approve/deny
- Persist rules to `egress_rules.json` (survives restart)
- Temporary rules tracked in memory with TTL

## Risk Assessment Heuristic
- 🟢 Green: domain in well-known safe list (api.openai.com, api.anthropic.com, github.com, icloud.com)
- 🟡 Yellow: unknown domain, standard ports (80/443)
- 🔴 Red: IP address (not domain), non-standard ports, known malicious TLDs, matches exfil patterns
