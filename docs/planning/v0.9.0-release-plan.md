# AgentShroud v0.9.0 — Release Plan
**Theme:** Multi-Runtime & Multi-Platform  
**Target:** Q3 2026

## Features

### 1. Multi-Runtime Testing (Podman + Apple Containers)
**Priority:** P1 | **Effort:** ~15h

AgentShroud must run and pass tests on all container runtimes across the fleet.

**Targets:**
| Host | Runtime | Arch | Python | Status |
|------|---------|------|--------|--------|
| Marvin | Colima/Docker | arm64 | 3.13 (Docker), 3.14 (host) | ✅ Working |
| Trillian | Docker/Colima | x86_64 | 3.14 (host) | ❌ Untested |
| Pi | Docker/Podman | arm64 | 3.9 (host) | ❌ Untested |

**Deliverables:**
- [ ] `Containerfile` (Podman-compatible) alongside `Dockerfile`
- [ ] Apple Containers support (macOS 26+) — evaluate `container` CLI
- [ ] CI matrix: Docker (arm64, x86_64) + Podman (arm64) + Apple Containers
- [ ] `make test-all-runtimes` target
- [ ] Resource-constrained test profile for Pi (skip ClamAV/Trivy on <4GB RAM)
- [ ] Document minimum system requirements per runtime
- [ ] Automated cross-host test runner (SSH + report aggregation)

### 2. Multi-Host Deployment Orchestration
**Priority:** P1 | **Effort:** ~20h

Coordinated deployment and testing across Marvin, Trillian, and Raspberry Pi.

**Deliverables:**
- [ ] Deployment playbook per host (Docker, Podman, Apple Containers)
- [ ] Centralized test runner: SSH into each host, run suite, aggregate results
- [ ] Per-host Telegram bot validation

### 3. ARM32 / Low-Resource Support
**Priority:** P2 | **Effort:** ~10h

- [ ] Lightweight Docker image variant (no ClamAV/Trivy/spaCy)
- [ ] Minimum viable security profile for <2GB RAM devices

### 4. Compliance & Standards Alignment
**Priority:** P2 | **Effort:** ~40h (research + implementation)
**Theme:** Enterprise and OT Environment Readiness

Investigation into how AgentShroud maps to global security standards across Enterprise IT,
OT/ICS, and container/cloud domains. The following standards have been identified as
priority targets for assessment and gap analysis.

#### 4a. OT / Industrial Security (P1 — Critical)
- [ ] **ISA/IEC 62443 (3-3, 4-1, 4-2)** — Map AgentShroud security controls to IEC 62443
  Foundational Requirements (FR1-FR7). Identify gaps for BESS/energy OT deployments.
  - FR1: Identification & Auth Control
  - FR2: Use Control (RBAC/egress firewall mapping)
  - FR3: System Integrity (tool-call inspection, input validation)
  - FR4: Data Confidentiality (PII redaction, credential isolation)
  - FR5: Restricted Data Flow (egress firewall, network segmentation)
  - FR6: Timely Response to Events (audit logging, owner alerts)
  - FR7: Resource Availability
- [ ] **NERC CIP (CIP-002 through CIP-015)** — Assess relevance for BES-connected BESS deployments.
  CIP-003-9 effective April 1, 2026. CIP-015 (Internal Network Security Monitoring) pending.
- [ ] **NIST SP 800-82 Rev. 3** — OT security baseline; Zero Trust for OT controls mapping.

#### 4b. Enterprise IT (P2 — High)
- [ ] **NIST CSF v2.0** — Map 6 core functions (Govern, Identify, Protect, Detect, Respond,
  Recover) to AgentShroud capabilities. Identify coverage gaps.
- [ ] **ISO/IEC 27001:2022** — ISMS control mapping. Required for enterprise customer contracts.
- [ ] **CIS Controls v8.1** — Evaluate IG1/IG2 controls applicable to agent security proxy.

#### 4c. Container & Cloud Security (P2 — High)
- [ ] **NIST SP 800-190** — Application Container Security Guide gap analysis against current
  Dockerfile, runtime config, and image pipeline.
- [ ] **CIS Docker Benchmark** — Run `docker-bench-security` against AgentShroud containers.
  Remediate all Level 1 findings. Document Level 2 deviations.
- [ ] **NIST SP 800-218 (SSDF)** — Secure Software Development Framework mapping for CI/CD
  pipeline and container build process.
- [ ] **SLSA Level 2** — Supply chain integrity: provenance for container images. Implement
  Cosign signing + Syft SBOM generation in CI pipeline.

#### 4d. Regulatory Compliance (P3 — Medium)
- [ ] **EU NIS2 Directive** — Assess EMEA deployment requirements: incident reporting timeline
  (24hr), governance documentation, supply chain security obligations.
- [ ] **EU Cyber Resilience Act (CRA)** — Vulnerability reporting obligations start Sept 2026.
  Assess whether AgentShroud qualifies as a "product with digital elements."
- [ ] **Australia SOCI Act** — ISA/IEC 62443 nationwide adoption (2025). Assess for APAC
  BESS deployment compliance.

#### 4e. Tooling Integration (P3 — Medium)
- [ ] **OpenSCAP + SCAP Content** — Integrate compliance scanning into CI pipeline.
  Targets: CIS Docker Benchmark, NIST 800-190 profile.
- [ ] **Trivy** — Add to CI pipeline for image vulnerability scanning (FR3 coverage).
- [ ] **Syft + Cosign** — SBOM generation and image signing for supply chain integrity.
- [ ] **Falco** — Evaluate runtime anomaly detection for container workloads (FR6).
- [ ] **Semgrep** — SAST integration for gateway Python codebase (IEC 62443-4-1 SDL).

#### 4f. Documentation Deliverables
- [ ] IEC 62443 FR mapping table (AgentShroud control → FR requirement)
- [ ] NIST CSF v2.0 coverage heat map
- [ ] CIS Docker Benchmark results report
- [ ] SBOM for gateway and bot images (Syft CycloneDX format)
- [ ] Compliance gap register with remediation priorities
